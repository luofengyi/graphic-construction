# MERC-main 服务器补文件与训练指南

这份文档用于在另一台服务器 `git clone` 本仓库后，快速补齐运行所需文件，并直接开始训练。

## 1. 拉取代码

```bash
git clone https://github.com/luofengyi/graphic-construction.git
cd graphic-construction/JOYFUL
```

## 2. 需要手动补充的文件

由于仓库已排除大文件（数据集和 PT），你需要把以下文件手动上传到服务器对应路径：

### 必需数据文件（训练必须）

- `JOYFUL/data/iemocap/data_iemocap.pkl`
- `JOYFUL/data/iemocap_4/data_iemocap_4.pkl`

目录结构必须是：

```text
JOYFUL/
  data/
    iemocap/
      data_iemocap.pkl
    iemocap_4/
      data_iemocap_4.pkl
```

### 可选模型文件（仅评估或断点续训需要）

- `JOYFUL/model_checkpoints/iemocap_4_best_dev_f1_model_atv.pt`

> 从头训练（`--from_begin`）不依赖该 PT 文件。

## 3. 创建与安装环境（推荐）

```bash
conda create -n joyful python=3.9 -y
conda activate joyful

# 先装 PyTorch 官方 cu121 套件（CUDA 12.2 驱动可兼容）
pip install --no-cache-dir \
  torch==2.1.2+cu121 torchvision==0.16.2+cu121 torchaudio==2.1.2+cu121 \
  --index-url https://download.pytorch.org/whl/cu121

# 基础与 NLP 依赖（锁定兼容版本）
pip install --no-cache-dir \
  numpy==1.26.4 pandas pydantic scipy scikit-learn tqdm \
  transformers==4.38.2 accelerate==0.27.2 sentence-transformers==2.6.1 tokenizers==0.15.2

# 图学习依赖（与 torch 2.1.2 配套）
pip install --no-cache-dir torchdata==0.7.1 dgl==2.1.0
pip install --no-cache-dir torch-geometric pygcl
pip install --no-cache-dir torch-sparse -f https://data.pyg.org/whl/torch-2.1.0+cu121.html
pip install --no-cache-dir torch-scatter -f https://data.pyg.org/whl/torch-2.1.0+cu121.html
```

## 5. 可直接训练的脚本

下面脚本可直接保存为 `run_train.sh` 后执行（或逐行执行）。

```bash
#!/usr/bin/env bash
set -e

cd "$(dirname "$0")/JOYFUL"

# 四分类 + 你当前实现的单路融合构图（binary + hyperedge expansion）
python train.py \
  --dataset iemocap_4 \
  --modalities atv \
  --graph_mode hybrid_expand \
  --sim_metric cosine \
  --sim_threshold 0.7 \
  --sim_topk 5 \
  --hyper_min_size 3 \
  --hyper_max_size 8 \
  --max_hyperedges_per_dialog 30 \
  --hyper_edge_ratio_cap 1.0 \
  --from_begin \
  --epochs 50 \
  --batch_size 32 \
  --device cuda:0
```

执行：

```bash
bash run_train.sh
```

## 6. 最小验证命令（先跑通）

建议先用 1 epoch 烟测：

```bash
python train.py --dataset iemocap_4 --modalities atv --graph_mode hybrid_expand --from_begin --epochs 1 --device cuda:0
```

看到 `Epoch`、`Dev set`、`Test set` 指标输出即表示训练流程正常。
